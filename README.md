# Quant Horizon API

FastAPI backend for quantitative market forecasts, session-by-session analysis,
financial backtests, and simulated trade acceptances. The application does not
send brokerage orders.

## Project structure

```text
quant_horizon/
├── db/
│   └── .gitkeep
├── src/
│   └── quant_horizon/
│       ├── api.py
│       ├── app.py
│       ├── cli.py
│       ├── config.py
│       ├── dtos.py
│       ├── entities.py
│       ├── investment_models.py
│       ├── market_calendar.py
│       ├── persistence.py
│       ├── pipeline.py
│       ├── security.py
│       └── services.py
├── tests/
├── .env.example
├── .gitignore
├── pyproject.toml
└── poetry.lock  # generated by `poetry install` or `poetry lock`
```

The application uses a `src` layout. Internal modules use package-relative
imports, which avoids accidental imports from files in the repository root.

## Requirements

- Python 3.11, 3.12, or 3.13
- Poetry 2.x

Check the installed versions:

```powershell
py --version
poetry --version
```

## Install with Poetry

From the project root:

```powershell
poetry config virtualenvs.in-project true
poetry install
```

The first command is optional. It makes Poetry create the virtual environment in
`.venv`, inside the project directory.

To inspect the environment:

```powershell
poetry env info
poetry show
```

## Environment variables

Create a local `.env` from the example:

```powershell
Copy-Item .env.example .env
```

The `.env` file is ignored by Git. Do not commit API keys or production
credentials.

Default database files:

```text
db/quant_horizon.sqlite3
db/quant_horizon_cache.sqlite3
```

The application creates the `db` directory automatically. The database files are
ignored by Git, while `db/.gitkeep` preserves the directory in the repository.

## Run locally

Recommended command:

```powershell
poetry run quant-horizon-api --reload --env-file .env
```

Equivalent Uvicorn command:

```powershell
poetry run uvicorn quant_horizon.api:app --host 127.0.0.1 --port 8000 --reload --env-file .env
```

The point of entry is `quant_horizon.api:app`, because `api.py` imports the
FastAPI instance and registers all routes.

- Documentation: `http://127.0.0.1:8000/docs`
- Health check: `http://127.0.0.1:8000/health`

## Run the forecasting pipeline directly

```powershell
poetry run python -m quant_horizon.pipeline --ticker AAPL --start 2010-01-01 --current_forecast_only
```

## Dependency management

Add a runtime dependency:

```powershell
poetry add package-name
```

Remove a dependency:

```powershell
poetry remove package-name
```

Install exactly the versions recorded in `poetry.lock`:

```powershell
poetry sync
```

Update compatible dependency versions and refresh the lock file:

```powershell
poetry update
```

Validate the project configuration:

```powershell
poetry check
```

The first `poetry install` or `poetry lock` command generates `poetry.lock`.
Commit both `pyproject.toml` and `poetry.lock` to GitHub. The lock file makes
local, CI, and deployment installations reproducible.

## API 2.0.0 routes

- `POST /v1/forecasts/current`
- `GET /v1/forecasts/current/{ticker}`
- `POST /v1/forecasts/historical`
- `POST /v1/forecasts/daily`
- `POST /v1/analyses/daily`
- `POST /v1/backtests/period`
- `POST /v1/backtests`
- `GET /v1/positions/{ticker}`
- `POST /v1/positions/{ticker}/acceptances`
- `DELETE /v1/positions/{ticker}/acceptances`
- `GET /v1/data/{ticker}`

## Model selection

Each request may set `model` to one of:

- `lightgbm`
- `catboost`
- `xgboost`
- `logistic`
- `random_forest`
- `ensemble`

When a request omits the field, `QUANT_HORIZON_MODEL` defines the server default.
If the variable is not defined, LightGBM is used.

## SQLite deployment

For a platform with a persistent volume mounted at `/data`, configure:

```text
QUANT_HORIZON_DB=/data/quant_horizon.sqlite3
QUANT_HORIZON_CACHE_DB=/data/quant_horizon_cache.sqlite3
```

A production start command can be:

```bash
poetry run uvicorn quant_horizon.api:app --host 0.0.0.0 --port $PORT
```

Do not store production SQLite files inside the ephemeral application filesystem.
Use a persistent volume or migrate the database to PostgreSQL when horizontal
scaling becomes necessary.

## Optional API key

Set `QUANT_HORIZON_API_KEY` before starting the server. Clients must send the
same value in the `X-API-Key` header.
